## Overview

The "Assignment Created" trigger is a workflow initiator that activates when a new assignment is created in the pipeline. This trigger serves as the starting point for automated processes, allowing you to set up actions that occur immediately after an assignment is generated. By utilizing this trigger, you can streamline your workflow and ensure consistent handling of new assignments from the moment they enter your system.The "Pipeline Changed" trigger is a powerful workflow initiator that activates when an item's pipeline stage is modified or moved. This trigger serves as the starting point for your workflow, automatically detecting changes in an item's status within your pipeline and enabling you to set up responsive actions based on these transitions.
